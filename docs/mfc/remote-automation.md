# <a name="remote-automation"></a>リモート オートメーション
> [!NOTE]
>  リモート オートメーションは、廃止されたテクノロジで新しいアプリケーションは使用できません。 Visual C では、Windows 95 がサポートされていません。 リモート オートメーションをサポートする必要があるとする場合は「[ここではリモート オートメーション合わせての](where-does-remote-automation-fit-in-q.md)します。  
  
 リモート オートメーションの型とは、[オートメーション](../mfc/automation.md)インターフェイス コンシューマーをネットワーク上などの別のコンピューターに置かれているインターフェイス プロバイダーを実行することができます。  
  
 この記事を起動およびリモートで実行されるオートメーション オブジェクトを作成する方法とリモート オートメーション オブジェクトを使用できるオートメーション コント ローラーを作成する方法について説明します。 また、設定オプションを確認し、主なリモート オートメーションおよび DCOM (COM との起動し、リモートで実行するように automation に関連するもの以外のインターフェイスは、OLE の分散型バージョン) 間の相違点をポイントします。
