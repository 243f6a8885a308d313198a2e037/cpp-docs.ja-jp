# [ユーザー インターフェイスの機能] (MFC アプリケーション ウィザード)
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

このトピックでは、アプリケーションの外観を指定するために使用できるオプションについて説明します。  プロジェクトで使用できるユーザー インターフェイス機能は、MFC アプリケーション ウィザードの [\[アプリケーションの種類\] \(MFC アプリケーション ウィザード\)](../Topic/Application%20Type,%20MFC%20Application%20Wizard.md) ページで指定したアプリケーションの種類によって異なります。  たとえば、シングル ドキュメント インターフェイス アプリケーションを作成する場合は、子フレームのスタイルは追加できません。  
  
 **\[メイン フレームのスタイル\]**  
 アプリケーションのメイン ウィンドウ フレームの機能を設定します。  
  
|オプション|説明|  
|-----------|--------|  
|**\[サイズ変更\]**|サイズ変更境界を使用するウィンドウが作成されます。  これが既定値です。|  
|**\[最小化ボタン\]**|メイン フレーム ウィンドウに最小化ボタンが組み込まれます。  これが既定値です。|  
|**\[最大化ボタン\]**|メイン フレーム ウィンドウに最大化ボタンが組み込まれます。  これが既定値です。|  
|**\[最小化\]**|メイン フレーム ウィンドウがアイコンとして表示されます。|  
|**\[最大化\]**|メイン フレーム ウィンドウがディスプレイの最大サイズで表示されます。|  
|**\[システム メニュー\]**|メイン フレーム ウィンドウにシステム メニューが組み込まれます。  これが既定値です。|  
|**\[バージョン情報\]**|アプリケーションの **\[バージョン情報\]** ボックスが組み込まれます。  このボックスへは、アプリケーションの **\[ヘルプ\]** メニューからアクセスします。  これは既定値であり、[\[アプリケーションの種類\] \(MFC アプリケーション ウィザード\)](../Topic/Application%20Type,%20MFC%20Application%20Wizard.md) ページの **\[ダイアログ ベース\]** を選択しない限り変更できません。<br /><br /> **メモ** 通常、オプションが使用できないということは、そのオプションのチェック ボックスがオンでもオフでも、ウィザードがそのオプションをプロジェクトに適用しないことを意味します。  この場合、プロジェクトをダイアログ ベースに指定してからチェック ボックスをオフにしない限り、常にウィザードによって **\[バージョン情報\]** ボックスがプロジェクトに追加されます。|  
|**\[初期ステータス バー\]**|アプリケーションにステータス バーが追加されます。  ステータス バーには、キーボードの **CapsLock** キー、**NumLock** キー、**ScrollLock** キーの自動インジケーターと、メニュー コマンドやツール バー ボタンのヘルプ文字列を表示するメッセージ行が含まれます。  このオプションをクリックして、ステータス バーを表示または非表示にするメニュー コマンドを追加することもできます。  既定では、アプリケーションにはステータス バーが含まれます。  ダイアログ ベースのアプリケーションには使用できません。|  
|**\[分割ウィンドウ\]**|分割バーを設定します。  分割バーによってアプリケーションのメイン ビューが分割されます。  マルチ ドキュメント インターフェイス \(MDI: Multiple Document Interface\) アプリケーションでは、MDI 子フレームのクライアント ウィンドウが分割ウィンドウです。シングル ドキュメント インターフェイス \(SDI: Single Document Interface\) アプリケーションおよびマルチ トップレベル ドキュメント アプリケーションでは、メイン フレームのクライアント ウィンドウが分割ウィンドウです。  ダイアログ ベースのアプリケーションには使用できません。|  
  
 **\[子ウィンドウのフレーム\]**  
 アプリケーションの子フレームの表示状態と初期状態を指定します。  子フレームのスタイルを指定できるのは MDI アプリケーションだけです。  
  
|オプション|説明|  
|-----------|--------|  
|**\[最小化ボタン\]**|子ウィンドウに最小化ボタンを表示するかどうかを指定します \(既定では有効\)。|  
|**\[最大化ボタン\]**|子ウィンドウに最大化ボタンを表示するかどうかを指定します \(既定では有効\)。|  
|**\[最大化\]**|`CChildFrame` のメンバー関数 [PreCreateWindow](../Topic/CWnd::PreCreateWindow.md) に cs.style フラグ **WS\_MAXIMIZE** を設定することにより、初期状態で子ウィンドウを最大化するかどうかを指定します。|  
  
 **\[コマンド バー \(メニュー\/ツール バー\/リボン\)\]**  
 アプリケーションにメニュー、ツール バー、またはリボンを組み込むかどうかを指定します。  ダイアログ ベースのアプリケーションには使用できません。  
  
|オプション|説明|  
|-----------|--------|  
|**\[クラシック メニューを使用する\]**|アプリケーションに、従来のドラッグできないメニューを組み込むよう指定します。|  
|**\[クラシックのドッキング可能ツール バーを使用する\]**|アプリケーションに標準の Windows ツール バーが追加されます。  ツール バーには、新規ドキュメントの作成、ドキュメント ファイルのオープンや保存、テキストの切り取り、コピー、貼り付け、または印刷、およびヘルプ モードへの切り替えのためのボタンが含まれます。  このオプションを有効にすると、ツール バーを表示または非表示にするメニュー コマンドを追加することもできます。|  
|**\[ブラウザー スタイルのツール バーを使用する\]**|アプリケーションに Internet Explorer 形式のツール バーが追加されます。|  
|**\[メニュー バーとツール バーを使用する\]**|アプリケーションに、ドラッグ可能なメニュー バーとツール バーを組み込むよう指定します。|  
|**\[ユーザー定義のツール バーとツール バー イメージ\]**|ユーザーが実行時にツール バーとツール バーのイメージをカスタマイズできるようにします。|  
|**\[パーソナル化されたメニュー動作\]**|メニューを開いたときに、すべての項目を一覧表示するか、ユーザーが最も頻繁に使用するコマンドだけを表示するかを指定します。|  
|**\[リボンを使用する\]**|アプリケーションでメニュー バーやツール バーの代わりに Office 2007 のようなリボンが使用されます。|  
  
 **\[ダイアログ タイトル\]**  
 [CDialog クラス](../../mfc/reference/cdialog-class.md)ベースのアプリケーションの場合にのみ、このタイトルがダイアログ ボックスのタイトル バーに表示されます。  このフィールドを編集するには、**\[アプリケーションの種類\]** の **\[ダイアログ ベース\]** をあらかじめ選択しておく必要があります。  詳細については、「[\[アプリケーションの種類\] \(MFC アプリケーション ウィザード\)](../Topic/Application%20Type,%20MFC%20Application%20Wizard.md)」を参照してください。
