# 2.9 Directive Nesting
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

ディレクティブの動的な入れ子は次の規則に従う必要があります :  
  
-   **並列**  のディレクティブによって別の  **並列**  内で論理的に入れ子になった並列化が有効になっていない場合は現在のスレッドだけで構成される新しいチームを確立します。  
  
-   **For** が **セクション**  と同じ  **並列**  にバインドする  **シングル**  のディレクティブどうしを入れ子にできません。  
  
-   同じ名前の  **重大**  のディレクティブが互いを入れ子にできません。  この制限にはデッドロックの防止十分注意してください。  
  
-   ディレクティブが領域と同じ  **並列**  にバインドする場合 **For セクション**  と  **シングル**  のディレクティブは **重大  順序あり**  と  **マスター**  の動的領域のスコープでは使用できません。  
  
-   **バリア**  のディレクティブは**For 順序あり  セクション  シングル  マスター**  と  **重大**  領域の動的範囲にディレクティブが領域と同じ  **並列**  にバインドすることはできません。  
  
-   **マスター**  のディレクティブは**For セクション**  と  **シングル**  ディレクティブのスコープで動的  **マスター**  のディレクティブのしくみ共有のディレクティブと同じ  **並列**  にバインドすることはできません。  
  
-   **順序あり**  のディレクティブは  **重大**  領域の動的範囲にディレクティブが領域と同じ  **並列**  にバインドすることはできません。  
  
-   並列領域の外部で実装された場合許可されているディレクティブは並列領域で動的に実行された場合は許可されます。  ユーザーが指定した並列領域外で動的に実行されるとディレクティブはマスター スレッドだけで構成されたチームによって実装されます。
